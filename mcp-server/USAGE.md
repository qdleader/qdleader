# QDLeader MCP 使用指南

## 快速开始

### 在 Cursor 中配置

1. 打开 Cursor 设置 (⌘+,)
2. 找到 "Features" → "MCP"
3. 点击 "Edit Config"
4. 添加以下配置：

```json
{
  "mcpServers": {
    "qdleader": {
      "command": "node",
      "args": ["/Users/icourt/github/qdleader/mcp-server/dist/index.js"]
    }
  }
}
```

### 重启 Cursor

配置完成后，需要重启 Cursor 以加载 MCP 服务器。

## 使用场景

### 场景 1: 学习 Vue3

```
我想学习 Vue3 的组合式 API，帮我搜索相关文档
```

AI 助手会：
1. 使用 `search_knowledge` 搜索 "Vue3 组合式 API"
2. 返回相关文档列表
3. 使用 `read_document` 读取具体文档
4. 为你讲解内容

### 场景 2: 准备面试

```
我要准备前端面试，帮我找 JavaScript 闭包相关的面试题
```

AI 助手会：
1. 在 "前端面试" 分类中搜索
2. 找到相关的面试题文档
3. 读取并为你讲解

### 场景 3: 解决问题

```
我在使用 TypeScript 时遇到泛型的问题，有相关文档吗？
```

AI 助手会：
1. 搜索 TypeScript 和泛型相关内容
2. 提供相关文档
3. 帮助你理解和解决问题

### 场景 4: 浏览分类

```
列出所有可用的技术分类
```

AI 助手会使用 `list_categories` 显示所有分类及文档数量。

### 场景 5: 系统学习

```
获取 React 分类下的所有文档
```

AI 助手会使用 `get_category_contents` 列出该分类的所有文档，让你系统学习。

## 最佳实践

### 1. 精确搜索

使用具体的关键词会得到更好的结果：

✅ 好的搜索：
- "Vue3 组合式 API"
- "React hooks useEffect"
- "TypeScript 泛型"
- "Webpack 配置优化"

❌ 不好的搜索：
- "Vue"（太宽泛）
- "前端"（太宽泛）

### 2. 利用分类过滤

如果知道要找的内容属于哪个分类，使用分类过滤：

```
在 vue3 分类中搜索响应式原理
```

### 3. 组合使用工具

先浏览分类，再查看具体内容：

```
1. 列出所有分类
2. 获取前端面试分类的内容
3. 读取具体的面试题文档
```

## 常见问题

### Q: MCP 服务器没有启动？

**A:** 检查以下几点：
1. 确认配置路径正确
2. 确认已经运行 `npm run build`
3. 重启 Cursor
4. 查看 Cursor 的开发者工具控制台是否有错误

### Q: 搜索不到内容？

**A:** 
1. 尝试使用不同的关键词
2. 使用 `list_categories` 查看可用的分类
3. 检查仓库是否有对应的文档

### Q: 文档读取失败？

**A:** 
1. 确认路径正确（使用 search_knowledge 返回的路径）
2. 确认文件存在且可读
